### BIOL550 Week 5: RNA Structure Prediction (Using BIO630 Algorithms)

**Methods Memo W5:**
- **Summary:** RNA secondary structure prediction (Nussinov algorithm from BIO630) identifies structured vs. unstructured regions; structured regions may be easier to measure, creating fairness risk if certain ancestry groups have more structured transcripts.
- **Failure modes:**
  1. **Structural heterogeneity bias:** if minority ancestry group has fewer structured transcripts, feature selection may over-weight unstructured features (which are noisier), reducing their signal
  2. **Measurement error coupling:** unstructured regions are harder to sequence; if sequencing depth differs by ancestry, unstructured features will be over-affected in minority groups
  3. **Complexity filtering:** if we exclude highly complex structures (>80th percentile complexity), we may exclude ancestry-specific regulatory elements
- **Project spine connection:** "We will *not* use purely structural complexity as a filter, because it may erase ancestry-specific isoforms. Instead, we stratify complexity analysis by ancestry before filtering."

**Code B (Week 5):**
- Implement Nussinov on 50 candidate biomarkers
- Output: complexity score per transcript
- Stratify by ancestry; visualize: ancestry 1 vs. ancestry 2 complexity distributions
- Decision: do distributions differ significantly? If yes, use ancestry-stratified complexity thresholds

**Spine Ledger Entry W5:**
- **Decision:** Structure-aware feature engineering with ancestry-stratified complexity thresholds
- **Rationale:** Preliminary analysis shows complexity distributions differ by ancestry. Using global threshold would disproportionately filter minority-ancestry transcripts.
- **Fairness impact:** Maintains equal representation of complex and simple features across ancestry groups
- **Trade-off:** Slightly increased feature noise from retaining more unstructured transcripts, but improved DI from 0.65 to 0.82
- **GitHub commit:** `/biol550/week5_rna_structure.py` + `/biol550/complexity_stratified_analysis.ipynb`
